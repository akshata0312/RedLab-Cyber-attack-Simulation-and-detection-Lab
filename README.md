# Aegis-Red: Local Attack Simulation & Detection Lab

Aegis-Red is a comprehensive Red Team cybersecurity simulation environment designed for internship portfolios. It allows users to simulate real-world attack techniques, observe system responses, and understand how defensive monitoring (SIEM) detects offensive actions.

## 🚀 Project Concept
This lab simulates a "Cyber Range" where an intern can act as a Red Team operator. By launching various attack modules, the system generates "System Logs" which are then parsed by a "Detection Engine" to trigger "Security Alerts."

## 🛠 Technical Stack
- **Frontend**: React 19, Tailwind CSS 4, Motion (Animations), Lucide Icons
- **Backend**: Node.js (Express), `better-sqlite3`
- **Database**: SQLite (Local file-based storage)
- **Security Logic**: Custom heuristic-based detection engine

## 📂 Project Structure
- `server.ts`: The core engine handling simulations, detections, and API routes.
- `src/App.tsx`: The high-fidelity dashboard for C2 (Command & Control) operations.
- `src/types.ts`: Shared data structures for logs, alerts, and modules.
- `aegis_red.db`: Local SQLite database (generated on first run).

## 🛡 Attack Modules
1. **SSH Brute Force**: Simulates high-frequency failed login attempts.
2. **Credential Harvesting**: Simulates phishing data entry.
3. **SQL Injection**: Simulates authentication bypass attempts.
4. **Log Tampering**: Simulates evasion by modifying audit trails.
5. **Privilege Escalation**: Simulates exploitation of local misconfigurations.
6. **Network Recon**: Simulates stealthy port scanning.

## 📊 Key Features
- **Real-time SIEM Feed**: Watch alerts trigger as you launch attacks.
- **Evasion Control**: Adjust the "Stealth" level of attacks to see if you can bypass the detection engine.
- **Executive Reporting**: Automatically generate a security assessment report based on lab activity.
- **Audit Logs**: Full visibility into the raw system logs generated by offensive actions.

## ⚖️ Security Ethics
**EDUCATIONAL USE ONLY.** This project is a simulation lab designed for local environments. It does not perform real network attacks and is intended strictly for learning offensive security patterns and defensive monitoring.
